# 照片质量标注器

## 概述

使用这个 skill 来判断照片是否适合进入干净、正式的人像或活动精选集；为照片标注通过/不通过原因和可检索内容标签；去除重复或近似重复照片；给照片打系统标签；并把最好的达标照片复制到 `精选` 或用户指定的文件夹。始终保留原始文件，不要改动原图像素。

## 工作流程

1. 盘点照片文件夹。
   - 识别图片文件，忽略 `.DS_Store` 等系统文件。
   - 把同名不同格式的文件视为同一张照片，例如 `DSC02139.JPG` 和 `DSC02139.ARW`。
   - 用 JPG、HEIC 等可直接查看的文件做视觉判断。若照片入选，默认一并复制同名 RAW 文件，除非用户另有要求。

2. 如果照片很多，先建立可视化检查面板。
   - 生成带文件名的联系表，或直接逐张查看图片。
   - 客观指标只能作为辅助，例如清晰度、曝光、过曝/欠曝裁切、尺寸和文件损坏检查；最终仍以人工视觉判断为准。

3. 标注可见内容。
   - 先识别人物、地点/场景、天气/光线、活动、物品、服装和画面风格等内容标签。
   - 只标注画面中能看出来的信息；不要凭空识别姓名、关系、职业、民族、宗教、健康状态等敏感或不可见信息。
   - 如果用户提供了人物名单或事件背景，可以使用用户提供的专属标签。

4. 先应用硬性门槛，再打分。
   - 任何硬性门槛不满足，都标为 `REJECT`。
   - 只有问题轻微或不确定，并且没有更好的重复照片时，才标为 `MAYBE`。
   - 只有满足全部人物、表情和技术要求时，才标为 `PASS`。

5. 对重复或近似重复照片分组。
   - 按相邻文件名、拍摄时间、同一场景、同一组人物和相似构图分组。
   - 每个重复组里，只保留质量最高的一张 `PASS` 照片。
   - 如果一组里没有照片达标，则不选择任何照片，除非用户明确要求保底选择。

6. 输出结果：复制、打标签或两者都做。
   - 如果目标文件夹不存在，先创建。
   - 不要修改或删除原始照片。
   - 如果要替换已有精选文件夹，只清理目标文件夹里的旧精选副本，绝不要清理源文件夹里的原图。
   - 复制入选的可浏览图片文件，并复制同名 RAW/源文件。
   - 如果用户要求给照片打标签，优先同时输出 CSV/JSON 标注表，因为系统标签跨平台和跨格式不稳定。
   - 如果既复制又打标签，默认给复制后的精选文件打 `精选`/`达标` 标签；只有用户明确要求时，才给源文件打标签。

7. 汇报结果。
   - 告知目标路径、入选文件名、数量、已应用标签，以及重要的排除原因。

## 硬性门槛

以下标准用于以人物为主的严格筛选，属于不可妥协的通过/不通过条件：

- **面向镜头**：所有主要人物都应正对镜头，或接近正对镜头。明显侧脸、看向别处、注意力不在镜头上的照片应淘汰。
- **眼睛睁开**：所有主要人物眼睛应睁开。眨眼、半闭眼明显像眨眼，或主要人物脸太小/太糊导致无法确认眼睛状态时，应淘汰。
- **没有多余动作**：除非用户明确要求抓拍或动作照，否则淘汰带有多余手势或进行中动作的照片。例如剪刀手、比心、指向、挥手、喝茶、演奏乐器、击鼓、朗诵/说话张嘴、整理衣服、过度倾斜或摆搞怪姿势。
- **姿态干净**：人物应自然站立或端坐，手部自然，没有尴尬的运动中姿势。
- **无遮挡**：脸部不能被手、物体、头发、其他人、家具或过重阴影明显遮挡。
- **裁切自然**：如果头、脚、手或关键主体被无意切掉，或人物贴边过紧，应淘汰。
- **清晰度达标**：脸部应清晰。运动模糊、对焦错误或相机抖动导致脸发软，应淘汰。
- **曝光达标**：严重欠曝、脸部过曝、重要区域高光溢出，或色偏导致肤色不自然，应淘汰。
- **背景干净**：如果杂物、穿帮物件或设备明显分散注意力，应淘汰。

## 达标照片评分

照片通过硬性门槛后，再在同组候选里排序：

- 技术质量，40 分：脸部对焦、曝光干净、噪点低、颜色自然、分辨率足够。
- 表情和准备状态，30 分：眼睛睁开、表情放松、嘴部自然、看向镜头、没有尴尬紧张感。
- 构图，20 分：空间均衡、水平/垂直线稳定、距离合适、边缘没有挤压主体。
- 入选价值，10 分：比附近相似照片更能代表这个人物、场景或时刻。

宁可选择一张不那么有戏剧性但完全满足硬性门槛的照片，也不要选择一张更有趣但有手势、闭眼或人物没看镜头的照片。

## 标注格式

当用户要求输出标注时，创建 CSV 或 JSON，包含以下字段：

- `file`：用于视觉判断的源文件名。
- `group_id`：重复照片或同场景分组编号。
- `status`：`PASS`、`MAYBE` 或 `REJECT`。
- `score`：`PASS` 或 `MAYBE` 使用 0-100 分；明显淘汰可留空或填 0。
- `selected`：只有同组最终入选的最佳照片标为 `yes`。
- `reason_codes`：简短原因码，用英文逗号分隔。
- `quality_tags`：筛选质量相关标签，例如 `精选`、`闭眼`、`模糊`。
- `content_tags`：画面内容标签，例如 `儿童`、`室内`、`茶室`、`古风服装`。
- `system_tags`：实际写入 Finder/Windows 的标签；没有写入则留空。
- `notes`：简短中文说明。

建议原因码：

- `not_facing_camera`：没有正对镜头。
- `eyes_closed`：闭眼或疑似眨眼。
- `extra_gesture`：有多余手势。
- `active_motion`：处于动作中。
- `awkward_posture`：姿态不自然。
- `face_occluded`：脸部被遮挡。
- `bad_crop`：裁切不佳。
- `soft_focus`：对焦偏软。
- `motion_blur`：运动模糊。
- `bad_exposure`：曝光或色彩问题。
- `distracting_background`：背景干扰。
- `duplicate_weaker`：重复组里的较弱照片。
- `best_of_group`：重复组里最佳照片。

## 系统标签

系统标签是可选输出，不要替代 CSV/JSON 标注表。标签适合在 Finder、资源管理器或照片管理软件里快速浏览；CSV/JSON 适合长期保存、跨系统迁移和复查筛选逻辑。

推荐标签：

- `精选`：最终入选照片。
- `达标`：通过硬性门槛，但不一定最终入选。
- `待定`：轻微问题或需要用户确认。
- `不达标`：未通过硬性门槛。
- `闭眼`：闭眼或疑似眨眼。
- `未看镜头`：没有正对镜头。
- `多余动作`：有手势、摆拍动作或进行中动作。
- `模糊`：对焦偏软或运动模糊。
- `曝光问题`：严重欠曝、过曝或色偏。
- `重复未选`：同组里质量较弱的重复照片。

## 内容识别标签

内容标签用于检索照片里“有什么”，应与质量标签分开保存。优先使用少量准确标签，不要给每张照片堆太多泛泛标签。建议每张照片选择 3-8 个内容标签。

人物标签：

- 人数：`单人`、`双人`、`三人`、`多人`、`合影`、`集体照`。
- 可见年龄段：`儿童`、`少年`、`成人`、`长者`。只在明显可见时标注。
- 角色/关系：`学生`、`老师`、`家人`、`朋友`、`工作人员`、`表演者`、`观众`。只在画面或用户背景明确时标注。
- 姿态：`站姿`、`坐姿`、`端坐`、`正面照`、`半身照`、`全身照`、`正式合影`。
- 表情：`微笑`、`自然表情`、`严肃表情`。不要过度解读情绪。
- 身份姓名：只有用户提供姓名或名单时，才使用 `人物-姓名` 这类专属标签；不要凭脸猜测身份。

地点和场景标签：

- 空间：`室内`、`室外`、`门口`、`走廊`、`庭院`、`街道`、`广场`、`舞台`、`教室`、`茶室`、`书房`、`展厅`、`餐厅`、`酒店`、`家中`。
- 自然环境：`公园`、`树林`、`山景`、`水边`、`湖边`、`海边`、`草地`、`花园`、`天空`。
- 建筑/空间风格：`中式空间`、`古风场景`、`传统文化场景`、`现代室内`、`木质背景`、`书架背景`、`圆形背景`。
- 地点名称：如果 EXIF、文件夹名、用户说明或画面文字明确，可添加具体地点，例如 `地点-上海`、`地点-某某学校`。

天气和光线标签：

- 天气：`晴天`、`阴天`、`多云`、`雨天`、`雪天`、`雾天`、`风天`。室内照片看不出天气时不要强行标注。
- 时间/光线：`白天`、`夜晚`、`清晨`、`傍晚`、`自然光`、`室内灯光`、`逆光`、`侧光`、`柔和光`、`强光`、`低光`。
- 氛围：`明亮`、`温暖色调`、`冷色调`、`暗调`。这些是视觉描述，不代表真实天气。

活动和主题标签：

- 活动类型：`课程`、`演出`、`排练`、`活动记录`、`仪式`、`聚会`、`旅行`、`参观`、`拍摄花絮`。
- 传统文化：`国学`、`汉服`、`茶艺`、`书法`、`朗诵`、`礼仪`、`传统服饰`、`传统乐器`。
- 动作内容：`合影留念`、`讲解`、`阅读`、`书写`、`品茶`、`击鼓`、`表演`、`摆拍`。如果动作导致不达标，同时加质量标签 `多余动作` 或 `active_motion`。

物品和服装标签：

- 服装：`汉服`、`中式服装`、`校服`、`礼服`、`便装`、`统一服装`、`浅色服装`。
- 道具/物品：`茶具`、`茶桌`、`书架`、`书本`、`篮子`、`鼓`、`乐器`、`灯笼`、`屏风`、`装饰画`、`水果`。
- 主体对象：`人物为主`、`环境为主`、`道具为主`、`建筑为主`。

画面和构图标签：

- 景别：`远景`、`中景`、`近景`、`特写`、`全身`、`半身`。
- 方向：`横图`、`竖图`。
- 构图：`居中构图`、`对称构图`、`环境人像`、`正式摆拍`、`抓拍`。
- 用途：`封面候选`、`宣传候选`、`留档`、`花絮`、`社交媒体候选`。

内容标签规则：

- 内容标签可以和质量标签同时存在，例如 `儿童` + `茶室` + `汉服` + `不达标` + `闭眼`。
- 内容标签要描述可见事实，避免推断敏感属性或主观评价。
- 优先使用稳定、短、可搜索的中文词；同一概念保持同一写法，例如始终用 `茶室`，不要混用 `茶房`、`茶空间`。
- 如果用户有自己的标签体系，优先沿用用户提供的词表。

给标签时遵循这些规则：

- 一张照片可有多个标签，例如 `不达标` + `闭眼` + `重复未选`。
- 同名 RAW 文件默认跟随 JPG/HEIC 的标签结果，但如果系统不支持给 RAW 稳定写标签，至少在 CSV/JSON 中记录 RAW 同名文件。
- 不要为了写标签而重编码图片或改变照片像素。
- 写入系统标签前，确认当前平台和文件格式是否支持；不支持时使用 CSV/JSON 或文件夹分类作为回退。

平台注意事项：

- **macOS**：Finder 标签通常写在文件扩展属性中，适合本机 Finder 浏览。可以优先使用已安装的命令行标签工具；没有可靠工具时，不要手写不确定的二进制扩展属性，改用 CSV/JSON 并说明可在 Finder 中手动批量加标签。
- **Windows**：JPG 的“标记/Tags”兼容性较好，PNG、HEIC、RAW 可能不支持或不同软件显示不一致。只有在确认文件格式和工具链可写入时才批量写入；否则使用 CSV/JSON、XMP sidecar 或文件夹分类。
- **跨系统项目**：把 CSV/JSON 标注表视为权威结果；系统标签只作为本机浏览辅助。

## 精选规则

- 严格人像或活动精选时，宁可少选，也不要放宽标准。
- 正式合影要求组内每个人都满足硬性门槛。
- 单人照片优先选择正对镜头、眼神明确、姿态简单对称的照片。
- 混合场景只保留主体明确、且所有主要人物都达标的照片。
- 如果用户的新要求和本 skill 冲突，遵循用户最新要求，并简短说明取舍。
